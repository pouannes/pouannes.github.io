Setting Up a First Personal Website

Jan 19, 2019 in MISC
webdev
4 min read

Here I’ll describe how I went about making my personal website (this one) and how I deployed it on github pages. The whole process wasn’t as easy as I thought it would be so hopefully this will help someone else down the line.

The assumptions I’ll make :

  1. You are reasonnably familiar with the command line ;
  2. You are reasonnably familiar with Git and GitHub ;
  3. You are reasonnably familiar with the web languages : HTML and CSS.

If you don’t fell comfortable enough with any of those there’s plenty of great tutorial out there. For example you can head over to codeacademy and then come back here.

What are we using to make our website

GitHub provide a very helpful and free service know as GitHub Pages. It offers users to host any static website for free. There’s plenty of different tools to create static websites. We’ll be using Hugo in this tutorial but the process is very similar with the other tools. You can find a list here. The proccess is a bit easier with Jekyll because it’s already integrated into GitHub pages, but I didn’t find any theme I liked so here we are.

1. Getting the website up and running locally using Hugo

  1. Install Hugo. On MacOs this can be done with

    brew install hugo
    For other OS and other ways of installing hugo see hugo’s documentation.

    You can verify your install with 

    hugo version

  2. Create a new site with :

    hugo new site <YOUR-WEBSITE-NAME>

    It’s that simple !

  3. Move to the repository you’ve just created :

    cd <YOUR-WEBSITE-NAME>

    You can now download a theme you like. You can browse hugo’s themes on this link. Once you’ve choosen a theme, do a git clone in the themes directory. For example with hyde-hyde, the theme used on this website :

    git init
git clone https://github.com/htr3n/hyde-hyde.git themes/hyde-hyde

    and add the name of the theme to the config.toml file of the theme. You can directly edit the file with a text editor or enter into the CLI (for hyde-hyde):

    echo ‘theme = “hyde-hyde”’ >> config.toml

  4. You can now customize your website. To start a local server that rebuilds everytime you change the file of your website, type

    hugo server -D

    The -D option makes the drafts visible.

    To add new content you can simply type

    hugo new posts/<NAME-OF-YOUR-POST>

Once you’re happy with the website you’ve built locally, it’s time to deploy it on GitHub Pages !

2. Deploy the website on GitHub Pages

  1. You’ll have to create two repositories : one for storing the hugo files, and another for storing your actual website. Careful, the repository for storing your actual website must be of the format <YOUR-GITHUB-USERNAME>.github.io. For example for me it’s pouannes.github.io

  2. Add the github remote origin to your local repo containing the hugo website and push your files. For example with :

    git remote add origin <GITHUB-REPO-URL>
git push -u origin master

  3. You can then build the hugo website simply by typing

    hugo

    This will create a build directory that contains your website. You don’t want git to track that directory with the rest of the files here so add it to .gitignore, for example with :

    echo “build” >> .gitignore

  4. Head to your public directory

    cd public

    And set your <YOUR-GITHUB-USERNAME>.github.io repo as the remote :

    git init
git remote add origin <GITHUBUSERNAME.GITHUB.IO-REPO-URL>
git push -u origin master

https://<YOUR-GITHUB-USERNAME>.github.io !

3. Automating future changes to your website

To recap, making change to your website locally involves testing the changes using hugo server -D, and then pushing the modifications to github (from the root of the website). Making change to how your site looks involves building the site with the command hugo and then pushing the changes from the /public directory. Here’s a little script to automatically commit and push the changes in your website, build the website, and commit and push the website. The script also asks you for the commit message for both pushes.

#!/bin/bash
git add .
echo “Enter commit message: “
read commitMessage
git commit -m “$commitMessage”
git push
hugo
cd public
git add .
git commit -m “$commitMessage”
git push

Put that script in a file like build.sh in the root of your project. You can make it executable with

chmod +x build.sh

and then execute it with

./build

You can also execute it simply with

sh build.sh
